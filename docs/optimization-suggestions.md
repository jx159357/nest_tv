# Nest TV 项目优化建议

## 一、功能优化

### 1.1 搜索体验优化

**现状**: 搜索使用 SSE 流式返回，但缺少搜索历史联想和热门搜索展示

**建议**:
- 搜索框增加防抖 (debounce 300ms)，输入时实时显示搜索建议
- 首页搜索区域展示热门搜索关键词（已有 API: `/search/popular-keywords`）
- 搜索结果增加"搜索历史"快捷入口
- 搜索结果支持按类型筛选（电影/电视剧/动漫等）

### 1.2 首页内容优化

**现状**: 首页有轮播、热门推荐、最新更新、高分推荐四个模块

**建议**:
- 增加"继续观看"模块（未看完的内容）
- 增加"猜你喜欢"个性化推荐模块（需登录）
- 各模块支持"查看更多"跳转
- 轮播增加自动播放控制按钮

### 1.3 播放器优化

**现状**: ArtPlayer + HLS.js + 弹幕叠加

**建议**:
- 增加画中画 (PiP) 模式支持
- 增加倍速播放记忆功能
- 增加手势控制（移动端左右滑动调进度，上下滑动调音量/亮度）
- 增加自动跳过片头/片尾功能
- 增加播放历史断点续播提示

### 1.4 用户体验优化

**建议**:
- 增加全局加载进度条（页面切换时）
- 增加骨架屏加载状态（首页、详情页等）
- 增加操作确认弹窗（删除等危险操作）
- 增加快捷键支持（空格暂停、方向键调进度等）

## 二、样式优化

### 2.1 设计系统统一

**现状**: 使用 CSS 变量定义了设计系统，但部分组件可能未完全遵循

**建议**:
- 统一所有组件的颜色使用 `var(--color-*)` 变量
- 统一间距使用 `var(--spacing-*)` 变量
- 统一圆角使用 `var(--radius-*)` 变量
- 统一阴影使用 `var(--shadow-*)` 变量

### 2.2 响应式优化

**建议**:
- 移动端导航栏改为底部 Tab 栏
- 媒体卡片网格自适应列数（手机1列、平板2列、桌面4列）
- 播放器在移动端自动全屏
- 管理后台侧边栏可折叠

### 2.3 暗色模式优化

**建议**:
- 确保所有组件在暗色模式下显示正常
- 图片在暗色模式下适当降低亮度
- 增加暗色模式切换动画过渡

### 2.4 动画优化

**建议**:
- 页面切换增加过渡动画
- 列表项增加 stagger 入场动画
- 弹窗/抽屉增加滑入/滑出动画
- 按钮增加按压缩放反馈

## 三、性能优化

### 3.1 前端性能

**建议**:
- 路由懒加载已实现，确保所有路由都使用懒加载
- 图片使用 `loading="lazy"` 和 `srcset` 响应式图片
- 大列表使用虚拟滚动（如搜索结果、媒体列表）
- API 响应使用内存缓存 + localStorage 缓存
- 减少不必要的组件重新渲染（合理使用 `computed` 和 `watch`）

### 3.2 后端性能

**建议**:
- 热门数据使用 Redis 缓存（媒体详情、频道列表等）
- 数据库查询添加必要索引
- API 响应启用 gzip 压缩
- 爬虫任务使用队列管理，避免阻塞主线程

---

## 四、安全优化

### 4.1 前端安全

**建议**:
- XSS 防护：用户输入内容使用 `v-text` 而非 `v-html`
- 敏感信息（如 token）不在 URL 中传递
- 增加 CSP (Content Security Policy) 配置

### 4.2 后端安全

**建议**:
- API 请求限流已实现，确保所有公开 API 都有限流
- 敏感操作（删除、修改）记录操作日志
- 密码强度验证
- 登录失败次数限制

---

## 五、代码质量优化

### 5.1 前端代码

**建议**:
- 统一错误处理机制（全局错误边界）
- TypeScript 类型定义完善
- 组件 Props 类型定义完整
- 减少组件内硬编码，提取为常量

### 5.2 后端代码

**建议**:
- 统一响应格式
- 统一错误处理
- DTO 验证完善
- 日志记录规范化
